# 한림병원 직원용 포탈 프로젝트 규칙 (Project Rules) v2.0

## 1. 기술 스택 및 아키텍처 (Tech Stack & Architecture)
- **Frontend**: 속도와 유지보수 효율을 위해 무거운 외부 프레임워크(React, Vue 등)를 배제하고, 단일 파일(`index.html`) 기반의 순수 자바스크립트(Vanilla JS)를 사용한다.
- **UI Framework**: Tailwind CSS (CDN 버전)를 사용하여 빠른 스타일링과 직관적인 클래스 관리를 수행한다.
- **Backend & DB**: Google Apps Script(GAS)를 백엔드 API로, Google Spreadsheet를 데이터베이스로 활용한다.
- **Data Fetching**: 프론트엔드의 데이터(공지, 인수인계, 장비 등)는 자바스크립트 객체 배열로 명확히 분리해두며, 추후 `fetch` API를 통해 GAS와 통신하여 데이터를 렌더링하기 쉬운 구조로 짠다.

## 2. 디자인 시스템 (Toss 스타일 지향)
- **UI/UX 방향성**: 토스(Toss) 앱처럼 직관적이고 여백이 넉넉하며, 부드러운 곡선과 은은한 그림자를 활용한 미니멀하고 모던한 룩을 지향한다.
- **배경 및 카드**: 전체 배경은 연한 쿨그레이(`#f2f4f6`), 콘텐츠 카드는 순백색(`#ffffff`)을 사용하여 영역을 부드럽게 분리한다.
- **컴포넌트 스타일**: 모든 버튼과 입력창, 카드는 둥근 모서리(`rounded-xl`, `rounded-2xl`)를 적용하고, 토스 특유의 옅은 테두리와 포커스 시 부드러운 전환 효과(`transition-all`)를 준다.
- **색상 및 시인성**: 병원 업무 환경을 고려해 메인 포인트 컬러는 메디컬 블루(`#004b8d`) 계열을 사용하고, 경고/알림은 오렌지(`#ff7a00`) 계열을 사용하여 시각적 피로도를 낮추고 가독성을 극대화한다.

## 3. 반응형 레이아웃 규칙 (Responsive Design)
- **PC 환경 (md 이상)**: 화면 좌측에 고정된 사이드바(Left Sidebar) 메뉴 구조를 적용하여 넓은 화면을 효율적으로 사용한다.
- **모바일 환경 (md 미만)**: 왼쪽 사이드바를 숨기고, 화면 하단에 고정된 네비게이션 바(Bottom Navigation Bar)를 탭 형식으로 배치하여 한 손 조작 편의성을 높인다.
- **모바일 방어적 CSS (오버플로우 방지)**: 좁은 모바일 화면에서 UI가 가로로 삐져나가거나 화면이 축소/확대되는 것을 막기 위해, Flexbox 사용 시 필요하다면 `flex-wrap`을 적용하고, Input 요소 등에는 `min-w-0`과 `w-full`을 적극 활용하여 유연하게 줄어들도록 설계한다.

## 4. 유연성 및 부서 확장성 (Scalability)
- **부서별 확장 구조**: 현재는 '기능검사실' 타겟이지만, 추후 병동, 외래, 수술실 등 타 부서가 추가될 수 있도록 시스템을 설계한다.
- **데이터 스키마**: 상태 객체(`state`)와 더미 데이터 배열을 만들 때, 반드시 `department`(부서) 혹은 `ward`(병동) 속성을 포함시킨다.
- **상태 관리**: 글로벌 상태 관리에 `currentDepartment` 상태를 두어, 로그인한 사용자의 소속 부서에 따라 UI 탭이나 데이터 필터링이 동적으로 변할 수 있도록 로직을 구성한다.

## 5. AI 코드 작성 주의사항 (Strict Constraints)
- **완전한 코드 제공**: 코드를 수정하거나 기능을 추가할 때 중간에 코드를 생략(`// 기존 코드 동일` 등)하지 말고, 항상 복사-붙여넣기로 즉시 실행 가능한 완전한 코드 블록을 제공한다.
- **UI/Data 분리**: UI를 그리는 함수(`render...`)와 데이터(`state...`)를 철저히 분리하여, 백엔드가 스프레드시트로 교체되더라도 HTML 구조는 건드리지 않게 작성한다.
